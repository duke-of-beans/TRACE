# TRACE Voice Guide

Adapted from the Tranche brand voice. Same philosophy: the facts create
the emotion. The copy does not.

## Core Rules

1. Never editorialize. The system records data. The operator draws conclusions.
2. No hero framing. Reporters are participants, not heroes.
3. No villain framing. "Subjects," "vehicles," "actors." Not "criminals,"
   "bad guys," "threats."
4. No security theater. State what the system does. Do not dramatize it.
5. Numbers over adjectives. "72 hours" not "extended period." "10 attempts"
   not "multiple failed attempts."
6. Short sentences. Active voice. Declarative.
7. Architecture over promises. "The key is destroyed" not "your data
   is safe." One is a fact. The other is a claim.
8. No mission language. Not "we protect" or "we believe." The system
   exists. It works. Here is what it does.
9. Let silence do the work. If a feature is good, naming it is enough.
   Do not explain why it is good.

## Forbidden Patterns

These are AI-associated or editorial patterns. None appear in TRACE copy.

### Punctuation
- Em dashes. Use periods, commas, or parentheses.
- Exclamation marks in UI copy.
- Ellipsis for emphasis.

### Words and Phrases
- "designed to" / "built to" / "crafted to"
- "seamlessly" / "effortlessly" / "robust" / "comprehensive"
- "leveraging" / "harnessing" / "empowering" / "enhancing"
- "cutting-edge" / "state-of-the-art" / "next-generation"
- "It's worth noting" / "Importantly" / "Notably"
- "ensures" / "guarantees" (architecture does not guarantee)
- "your journey" / "experience" / "adventure"
- "deep dive" / "unpack" / "explore" (as verbs for reading)
- "at the end of the day"
- "in order to" (use "to")
- "a wide range of" / "a variety of"
- "helps you" (the system does X, it does not "help you" do X)

### Structural
- Starting sentences with "This" as a vague referent.
- Compound sentences joined by em dash.
- Lists of three with escalating drama.
- Rhetorical questions.
- "Not just X, but Y" constructions.

### Framing
- "Protects you" (say what it does; protection is the user's conclusion)
- "Security-first" / "privacy-focused" (describe the architecture)
- "Your safety is our priority" (mission language)
- "In an emergency" (state the condition plainly)
- "Peace of mind" (emotional appeal)

## Voice Examples

### Bad (AI/editorial)
"TRACE is designed to help you report sightings quickly and securely,
ensuring your data remains protected at all times."

### Good (Tranche voice)
"TRACE records sightings. All data on the device is encrypted. The
encryption key is protected by your PIN."

### Bad
"Your chapter operator has powerful tools to protect you remotely,
including the ability to instantly wipe compromised devices."

### Good
"The operator can revoke access or destroy data on any device in
the chapter. This happens on the device's next server contact."

### Bad
"The Emergency Wipe permanently destroys all TRACE data on this
device, ensuring nothing can be recovered — even with physical access."

### Good
"Emergency Wipe destroys the encryption key, then clears all storage.
Without the key, stored data is unreadable. This cannot be undone."

## Context-Specific Rules

### Onboarding
Factual briefing. "The system does X." Not "We want you to know that X."
No welcome language beyond the first line. Get to the information.

### Security Documentation
Architecture descriptions. What the system does, how it is structured,
what the boundaries are. Never promise outcomes. Describe mechanisms.

### UI Labels
Shortest accurate phrase. "License Plate" not "Enter the vehicle's
license plate number." "Direction of Travel" not "Which direction
was the vehicle heading?"

### Error Messages
State what happened. State what to do. "[Specific reason]. [Action]."
Not "Oops! Something went wrong."

### Confirmation Dialogs
State the action and its consequence. "This revokes all sessions for
this reporter. They will need a new invite code." Not "Are you sure
you want to do this? This action cannot be undone!"

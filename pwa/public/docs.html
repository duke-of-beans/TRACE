Reporter workflow

Reporters use the phone app. No accounts with email or phone numbers - just a one-time invite code handed to them in person or via Signal.

What reporters can do

Submit sightings - plate, activity, location, photos. Takes 15 seconds. Reports queue offline and sync automatically.

Check plates - type a plate to check if it is already tracked. One-tap escalation to a full report.

Respond to dispatches - see operator-placed map pins. Tap to respond. Status flows back to the operator (responding, on scene, completed).

Report harassment - submit harassing phone numbers via the Alert tab. See operator responses and cross-reporter correlation.

Triage and dispatch

Incoming sightings land in the triage queue. Each one shows whether the plate matches a known vehicle or is new. Process them with keyboard shortcuts or click the action buttons.

Keyboard shortcuts speed up triage. Press ? to see all shortcuts. The essentials: C confirm and dispatch, D dismiss with feedback, F flag for follow-up, N / P next and previous.

Confirm and dispatch

When you confirm a sighting, choose an event type (configurable in Admin), set urgency (routine or urgent), and write a brief note for field reporters. A dispatch pin appears on every reporter's map.

Dismiss with feedback

Dismissing sends the reporter a brief explanation. This closes the feedback loop - reporters know their sighting was reviewed and why it was not acted on.

Activity Map

The intelligence layer. All sightings plotted on a map with multiple analysis overlays.

Triage Activity Map Dispatches
Heatmap Corridors Playback

Map layers

Heatmap shows concentration areas. Where are sightings clustering? Which intersections see the most activity?

Corridors trace a vehicle's movement path. Select a tracked vehicle to see every sighting location connected by route segments.

Time playback animates sightings hour by hour. Drag the slider to spot temporal patterns - a vehicle that appears every Tuesday at 3pm, for example.

Co-occurrence identifies vehicles that appear in the same area within the same time window. This surfaces potential associations.

Dispatch pins are placed by right-clicking the map. Reporters see these on their phones and can respond.

05

Vehicles and actors

Tracked vehicles and observed persons are the core intelligence records. Vehicles have concern levels that escalate based on evidence. Actors persist across vehicle retirements.

Concern levels

Vehicles progress through graduated concern levels as evidence accumulates. Promote a vehicle with a logged reason - the full from/to history is preserved.

NoticedFirst sighting, no pattern yet
MonitoringRepeat sightings, watching for pattern
Active ConcernClear pattern, active tracking
Priority WatchHigh-priority, immediate response
Concern levels and their names are fully configurable in Admin. Create your own ladder to match your chapter's terminology. Each level can have promotion predicates that define what evidence justifies escalation.

Vehicle detail view

Click any vehicle to see its complete record: plate, make, model, color, type assignments, concern history (last 10 promotions with reasons), recent sightings (last 20 with locations), and linked actors.

Actors

Observed person profiles with aliases (never real names), physical identifiers (scars, tattoos, build, clothing patterns), photos, and linked vehicles. An actor outlives their vehicles - when a car is retired, the person record persists and any new vehicle is linked to the same profile.

06

Incidents and evidence

When sightings become an investigation, create an incident. Link actors and vehicles, collect evidence through four phases, generate witness forms, and produce printable records.

Create incident → Link actors/vehicles → Collect evidence → Generate witness form → Close or escalate

Evidence phases

Evidence flows through four phases: Collection (gathering raw observations), Processing (organizing and tagging), Analysis (connecting patterns), and Presentation (printable observation record for authorities or community meetings).

Public witness forms

Generate a link to share with witnesses outside your reporter network. The link is token-gated (only works with the specific token), rate-limited (20 submissions max), and expires after 48 hours. Witnesses submit their account without needing an app or account.

Printable observation record

Every incident has a printable HTML record at /incidents/:id/record/print. Open it in a browser and print to PDF for a professional document suitable for authorities, insurance, or community meetings.

Evidence encryption. When EVIDENCE_ENCRYPTION_KEY is set in your Vercel environment variables, all evidence data is encrypted at rest using AES-256-GCM before writing to disk. The key should be 64 hex characters (256 bits).
07

Safety and security

TRACE is designed for situations where reporter safety matters. Multiple layers protect both the data and the people who submit it.

Three-vault architecture

Reporter identities (Vault B) are physically separated from operational data (Vault A). A full dump of the operations database reveals callsigns, never real names. Evidence sits in its own write-once vault (Vault C) with an independent encryption key.

Vault A - Ops
Vehicles, sightings, actors, dispatches
Zero real identities
Vault B - Identity
Real names, auth tokens, sessions
Encrypted, separate key
Vault C - Evidence
Write-once evidence locker
Hash chain, AES-256-GCM

Reporter safety features

PIN lock - the app requires a PIN to open. If someone picks up a reporter's phone, they cannot access sighting data.

Panic button - configurable in Settings. Triggers an emergency action defined by the operator. Accessible from any screen.

Dead man's switch - if a reporter does not open the app for 72 hours, local data is automatically wiped. No action required.

Offline queue - sightings submitted without connectivity are stored locally and synced when the connection returns. No data is lost.

Operator security controls

The Security page provides remote control over reporter devices: suspend (temporarily disable), kill (permanently disable), or nuke (wipe all data). These take effect the next time the reporter's app checks in.

What you can configure

Vehicle types - categories like Sedan, SUV, Pickup, Van. Each gets a color label. Assign multiple types to a vehicle.

Concern levels - your chapter's escalation ladder. Name them, set colors, define promotion predicates (what evidence justifies moving a vehicle up).

Dispatch event types - suspicious vehicle, welfare check, follow-up. Each has an icon, color, and auto-close timer (dispatches close automatically after the timer expires if not resolved).

Actor identifier types - field types for physical identifiers. Tattoo (text + body location), Hair Color (dropdown), Build (dropdown), etc. Dropdowns use a JSON options array.

Reporters - generate invite codes, view active reporters by callsign, see last check-in times.

Operators - create operator accounts with callsign + access code. Access codes work like passwords but are intentionally simple to share in person.

Integrations - CarAPI for automatic plate lookup (VIN, make, model, color). Spokeo for harassment phone number identification. Both optional, both cost per lookup.

Bug reports go directly to GitHub Issues. Click "Report Bug" in the sidebar to describe a problem. The report automatically sanitizes sensitive data (UUIDs, tokens) before creating the issue.

Production hardening

Before giving anyone access, set these environment variables in your Vercel dashboard (production only):

CORS_ORIGINS - your deployed URL (e.g. https://trace-abc123.vercel.app). Blocks cross-origin requests from other domains.

TRACE_DISABLE_DEV_LOGIN - set to true. Requires real access codes for operator login.

EVIDENCE_ENCRYPTION_KEY - 64 hex characters. Enables AES-256-GCM encryption for all evidence data at rest.

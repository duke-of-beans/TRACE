TRACE
Chapter Setup Guide
⏱ About 10 minutes · No coding required

TRACE is a vehicle tracking system for your neighborhood. Reporters use a phone app to submit sightings. You manage everything from a desktop dashboard.

To run TRACE, you need two things: a database (where your data lives) and a website (what people open on their phones and computers). Both are free, both are run by established companies, and you stay in full control.

1

Create your database

Why: Your database is where every sighting, vehicle, and dispatch record is stored. It's yours — no one else can see it. We use Neon because it's a trusted database company (used by thousands of businesses), it's free for small teams, and your data stays encrypted on their servers. You can export or delete everything at any time.

  1. Go to neon.tech and click Sign Up. You can use your Google account — or create a new email/password if you prefer not to link accounts.
  2. Click New Project. Name it anything — "trace" works fine. Pick the region closest to your area for faster performance.
  3. After it creates, you'll see a Connection String. It looks like gibberish — that's normal. Click the copy button next to it.

2

Deploy your TRACE website

Why: Your website is what reporters and operators open on their devices. Vercel hosts it — they run websites for companies like Nike, Uber, and The Washington Post, and their free tier is more than enough for a chapter. Your TRACE website is private to people who have your URL and an invite code.

Why GitHub: Vercel needs a GitHub account to work (GitHub stores the code, Vercel turns it into a website). Think of GitHub as the filing cabinet and Vercel as the storefront. You only need GitHub for this one step — you won't need to go back to it unless you want to update TRACE later.

  1. Click the blue Deploy button below. It opens Vercel.
  2. Sign in with GitHub. If you don't have one, click "Create an account" — it just needs an email and password. You won't need to learn GitHub.
  3. Vercel asks for DATABASE_URL. Paste the connection string you copied in Step 1.
  4. Click Deploy. Wait about 2 minutes while it builds your website.

3

Set up the database tables

Why: In Step 1, you created an empty database — like buying a filing cabinet with no folders inside. This step creates the folders (called "tables") that TRACE needs to organize sightings, vehicles, dispatches, and reporter accounts. You're pasting a set of instructions that tells the database how to organize itself.

You only do this once. It takes about 30 seconds.

  1. Go back to Neon and click your project.
  2. Click SQL Editor in the left sidebar.
  3. Open this file on GitHub. Click the copy raw file button (top-right of the code).
  4. Paste the entire thing into the Neon SQL Editor.
  5. Click Run.

CREATE SCHEMA IF NOT EXISTS "ops";

4

Create your operator account

Why: The operator is the person who manages the chapter — reviewing sightings, dispatching reporters, and tracking vehicles. TRACE doesn't use email addresses or real names for operator accounts. Instead, you choose a callsign (a code name like "ALPHA") and an access code (like a password). This protects your identity even if someone gains access to the system.

This screen only appears once — the first time you open the operator console. After that, it becomes a normal login screen.

  1. Go to your-app.vercel.app/operator/ (your Vercel URL + /operator/)
  2. You'll see a First-Time Setup screen.
  3. Enter your chapter name (e.g., "Westside Watch")
  4. Choose a callsign — this is your operator name (like "ALPHA" or "DISPATCH-1")
  5. Choose an access code — like a password, at least 6 characters. Keep this safe.
  6. Click Create Operator & Start

5

Invite your first reporter

Why invite codes: Reporters don't create accounts with email or phone numbers — that would put them at risk if the system were ever compromised. Instead, you generate a one-time invite code and hand it to them directly (in person or via encrypted chat like Signal). The code works once and expires in 7 days. No email, no phone number, no real name ever enters the system.

  1. In the operator console, click Admin in the left sidebar.
  2. Click the Reporters tab.
  3. Type a callsign for the reporter. This is their code name — not their real name. (e.g., "SCOUT-1")
  4. Click Generate Invite Code.
  5. You'll see a code like ABCD-EFGH. Send it to the reporter via Signal, WhatsApp, or in person.

Common Questions

It says "Connection failed" when I log in

Check that your DATABASE_URL is set correctly in Vercel. Go to your Vercel project → Settings → Environment Variables. The URL should start with postgresql://.

Reporter can't install the app on iPhone

Must use Safari. Chrome on iPhone does not support app installation. Open the URL in Safari, tap Share, then Add to Home Screen.

Invite code isn't working

Codes expire after 7 days and can only be used once. Generate a new one from Admin → Reporters.

How do I add more operators?

Go to Admin → Operators tab → Add Operator. Each operator gets their own callsign and access code.

How do I update TRACE when new versions come out?

In your GitHub repository, click "Sync fork" → "Update branch." Vercel automatically deploys the update. Your data in Neon is untouched — updates only change the app, not your records.

Your Accounts & Security

What accounts did I just create?

Neon — hosts your database. Your sightings, vehicles, and records live here. Free tier covers small-to-medium chapters. console.neon.tech

Vercel — hosts your website. This is what people open in their browser. Free tier covers most chapters. vercel.com/dashboard

GitHub — stores the code that Vercel builds from. You created this to deploy. You won't need to use it day-to-day. github.com

Is my data safe?

Yes. Your data is stored in your own private database on Neon's servers — encrypted at rest and in transit. Neon is SOC 2 Type II certified (an industry security standard). No one at Neon, Vercel, or GitHub can see your sighting data without your database connection string.

TRACE also separates reporter identities from sighting data at the architecture level. Even if someone got access to the sighting records, they couldn't determine who submitted them — that information is stored in a separate, isolated section of the database.

Can anyone find my TRACE website?

Your Vercel URL (like trace-abc123.vercel.app) is not listed anywhere publicly. It's a random URL that only people you share it with can find. Search engines won't index it. Even if someone discovers the URL, they can't do anything without an invite code (for the reporter app) or an operator callsign + access code (for the operator console).

How do I shut everything down and delete my data?

You're in full control. To completely remove TRACE:

1. Delete your Neon project — go to console.neon.tech, click your project → Settings → Delete Project. This permanently destroys all sighting data, vehicle records, and reporter accounts.

2. Delete your Vercel project — go to vercel.com/dashboard, click your project → Settings → scroll to bottom → Delete Project. Your website goes offline immediately.

3. (Optional) Delete the GitHub repo — go to github.com, open your TRACE repository → Settings → scroll to bottom → Delete this repository.

After these three steps, no trace of your chapter exists on any server. Reporter apps will stop working immediately and show an error screen.

Do any of these services cost money?

All three services (Neon, Vercel, GitHub) have generous free tiers. A chapter with up to ~50 reporters and thousands of sightings will stay within free limits. None of them require a credit card to sign up. If you ever approach a limit, the service will warn you before charging anything.

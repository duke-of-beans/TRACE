Chapter Setup Guide
About 10 minutes. No coding required.

TRACE is a vehicle tracking system for neighborhood chapters. Reporters submit sightings from a phone app. Operators review, dispatch, and track from a desktop console.

TRACE requires two services: a database (stores your records) and a web host (serves the app to devices). Both are free. Both use industry-standard encryption. You control both accounts and can delete everything at any time.

What you are setting up

TRACE is open-source software. The code is publicly available and can be reviewed by anyone. Nothing is installed on your computer. Instead, you create your own private copy running on your own cloud accounts.

The setup process creates three things:

1. A database on Neon (where your sighting records, vehicle data, and accounts are stored)

2. A website on Vercel (what reporters and operators open in their browser)

3. A code copy on GitHub (the source code that Vercel builds from)

All three are under your accounts. No one else has access. Your chapter's TRACE instance is completely separate from every other chapter's instance. There is no central server, no shared database, and no way for one chapter to see another's data.

1

Create the database

The database stores all sighting records, vehicle profiles, dispatch events, and accounts. Neon hosts PostgreSQL databases with encryption at rest (AES-256) and in transit (TLS 1.3). SOC 2 Type II certified. Free tier covers chapters with up to 50 reporters. Your database is isolated to your account. Neon staff cannot read your data without your connection credentials.

  1. Go to neon.tech and click Sign Up. Google account works, or use a separate email and password.
  2. Click New Project. Name it anything. Pick the region closest to your area.
  3. After creation, you will see a Connection String. It is a long string of characters. Click the copy button next to it.

2

Deploy the application

The Deploy button below does three things automatically: it copies the TRACE code to your GitHub account, connects it to Vercel, and builds your website. You do not need to download or install anything. The entire application runs in the cloud on your accounts.

Vercel serves the application over HTTPS. Each request runs in an isolated serverless function. Free tier supports thousands of daily requests.

  1. Click the Deploy button below. It opens Vercel.
  2. Sign in with GitHub. If you do not have an account, create one. It requires an email and password.
  3. Vercel asks for DATABASE_URL. Paste the connection string from Step 1.
  4. Click Deploy. Build takes about 2 minutes.

4

Create the operator account

The operator console is a website, not installed software. Open it in any browser on any computer. Nothing is stored on your machine. All data lives in the database from Step 1.

TRACE does not use email addresses or real names for accounts. Operators authenticate with a callsign (a code name) and an access code (a passphrase, 6+ characters). No identifying information enters the system. If the database were compromised, an attacker would find callsigns, not people.

This screen appears once, on first access. After the first operator is created, it becomes a standard login.

  1. Go to your-app.vercel.app/operator/ (your Vercel URL + /operator/)
  2. The First-Time Setup screen appears.
  3. Enter a chapter name (e.g. "Westside Watch")
  4. Choose a callsign (your operator code name, e.g. "ALPHA" or "DISPATCH-1")
  5. Choose an access code (passphrase, at least 6 characters). Store this securely.
  6. Click Create Operator & Start

5

Invite reporters

Reporters do not create accounts with email or phone numbers. The operator generates a one-time invite code and delivers it directly (in person or via encrypted channel). The code is single-use and expires in 7 days. No email, phone number, or real name enters the system at any point.

  1. In the operator console, click Admin in the left sidebar.
  2. Click the Reporters tab.
  3. Type a callsign for the reporter. Code name, not real name. (e.g. "SCOUT-1")
  4. Click Generate Invite Code.
  5. Deliver the code to the reporter via Signal or in person.

Common Questions

Connection failed on login

Verify DATABASE_URL in Vercel. Go to your Vercel project, Settings, Environment Variables. The value should start with postgresql://.

Reporter cannot install on iPhone

Must use Safari. Chrome on iPhone does not support PWA installation. Open the URL in Safari, tap Share, then Add to Home Screen.

Invite code not working

Codes expire after 7 days and can only be used once. Generate a new one from Admin, Reporters tab.

Adding more operators

Admin, Operators tab, Add Operator. Each operator gets their own callsign and access code.

Updating TRACE

In your GitHub repository, click "Sync fork," then "Update branch." Vercel deploys the update automatically. Data in Neon is unaffected.

Accounts and Security

What accounts were created

Neon stores the database. console.neon.tech

Vercel serves the application. vercel.com/dashboard

GitHub stores the source code. Not used after initial deployment. github.com

How data is stored

All data is in your Neon database. Neon encrypts at rest (AES-256) and in transit (TLS 1.3). SOC 2 Type II certified. Your database is isolated to your account.

TRACE separates reporter identities from sighting data at the database level. Sighting records, vehicle profiles, and dispatch events are in one schema. Reporter identity mappings are in a separate, isolated schema. A full export of the sighting data reveals zero real identities. The separation is structural, not policy.

Who can access the URL

The Vercel URL is not indexed by search engines or listed in any directory. The reporter app requires a valid invite code. The operator console requires a callsign and access code. An unauthenticated visitor sees a login screen and nothing else.

Shutting down and deleting all data

Three steps remove all data from all servers:

1. Delete the Neon project at console.neon.tech. Project, Settings, Delete Project. Destroys all records permanently.

2. Delete the Vercel project at vercel.com/dashboard. Project, Settings, scroll to bottom, Delete Project. The application goes offline immediately.

3. Delete the GitHub repository (optional). Repository, Settings, scroll to bottom, Delete this repository.

After these steps, no data from the chapter exists on any server. Reporter apps stop functioning immediately.

Cost

All three services (Neon, Vercel, GitHub) have free tiers. A chapter with up to 50 reporters and thousands of sightings stays within free limits. No credit card required. Each service warns before any charges.

Chapter Setup Guide
⏱ About 10 minutes · No coding required

TRACE is a vehicle tracking system for neighborhood chapters. Reporters submit sightings from a phone app. Operators review, dispatch, and track from a desktop console.

TRACE requires two services: a database (stores your records) and a web host (serves the app to devices). Both are free. Both use industry-standard encryption. You control both accounts and can delete everything at any time.

1

Create your database

The database stores all sighting records, vehicle profiles, dispatch events, and reporter accounts. Neon hosts PostgreSQL databases with encryption at rest and TLS in transit. The free tier covers chapters with up to 50 reporters. Your database is isolated to your account. Neon staff cannot read your data without your connection credentials.

  1. Go to neon.tech and click Sign Up. Google account works, or use a separate email/password.
  2. Click New Project. Name it anything. Pick the region closest to your area.
  3. After creation, you will see a Connection String. It is a long string of characters. Click the copy button next to it.

2

Deploy the TRACE application

Vercel serves the TRACE application to browsers and phones. All traffic is encrypted via HTTPS. The application runs in isolated serverless functions with no shared state between chapters. The free tier supports thousands of daily requests.

Vercel requires a GitHub account. GitHub stores the application code. Vercel compiles and serves it. You will not need to use GitHub after this step unless you update TRACE.

  1. Click the Deploy button below. It opens Vercel.
  2. Sign in with GitHub. If you do not have an account, create one. It requires an email and password.
  3. Vercel asks for DATABASE_URL. Paste the connection string you copied in Step 1.
  4. Click Deploy. Wait about 2 minutes while it builds your website.

3

Create the database tables

The database from Step 1 is empty. This step creates the table structure TRACE uses to organize sightings, vehicles, dispatches, and accounts. You paste one file into the Neon SQL editor and run it. One time only. Takes 30 seconds.

  1. Go back to Neon and click your project.
  2. Click SQL Editor in the left sidebar.
  3. Open this file on GitHub. Click the copy raw file button (top-right of the code).
  4. Paste the entire thing into the Neon SQL Editor.
  5. Click Run.

CREATE SCHEMA IF NOT EXISTS "ops";

4

Create the operator account

TRACE does not use email addresses or real names for accounts. Operators authenticate with a callsign (a code name) and an access code (a passphrase, 6+ characters). No identifying information enters the system. If the database were compromised, an attacker would find callsigns, not people.

This screen appears once, on first access. After the first operator is created, it becomes a standard login.

  1. Go to your-app.vercel.app/operator/ (your Vercel URL + /operator/)
  2. You'll see a First-Time Setup screen.
  3. Enter your chapter name (e.g., "Westside Watch")
  4. Choose a callsign (your operator code name, e.g. "ALPHA" or "DISPATCH-1")
  5. Choose an access code (passphrase, at least 6 characters). Store this securely.
  6. Click Create Operator & Start

5

Invite reporters

Reporters do not create accounts with email or phone numbers. The operator generates a one-time invite code and delivers it directly (in person or via encrypted channel). The code is single-use and expires in 7 days. No email, phone number, or real name enters the system at any point.

  1. In the operator console, click Admin in the left sidebar.
  2. Click the Reporters tab.
  3. Type a callsign for the reporter. This is their code name, not their real name. (e.g., "SCOUT-1")
  4. Click Generate Invite Code.
  5. You'll see a code like ABCD-EFGH. Send it to the reporter via Signal, WhatsApp, or in person.

Common Questions

It says "Connection failed" when I log in

Check that your DATABASE_URL is set correctly in Vercel. Go to your Vercel project → Settings → Environment Variables. The URL should start with postgresql://.

Reporter can't install the app on iPhone

Must use Safari. Chrome on iPhone does not support app installation. Open the URL in Safari, tap Share, then Add to Home Screen.

Invite code isn't working

Codes expire after 7 days and can only be used once. Generate a new one from Admin → Reporters.

How do I add more operators?

Go to Admin → Operators tab → Add Operator. Each operator gets their own callsign and access code.

How do I update TRACE when new versions come out?

In your GitHub repository, click "Sync fork" → "Update branch." Vercel automatically deploys the update. Your data in Neon is untouched. updates only change the app, not your records.

Your Accounts & Security

What accounts were created?

Neon. database host. Sighting records, vehicle data, and reporter accounts are stored here. Free tier covers small-to-medium chapters. console.neon.tech

Vercel. web host. Serves the application to browsers and phones. Free tier covers most chapters. vercel.com/dashboard

GitHub. code repository. Stores the application source that Vercel compiles. Not used after initial deployment. github.com

How is the data stored?

All data is stored in your Neon database. Neon encrypts data at rest (AES-256) and in transit (TLS 1.3). Neon holds SOC 2 Type II certification. Your database is isolated to your account. Neon cannot access your data without your connection credentials.

TRACE separates reporter identities from sighting data at the database level. Sighting records, vehicle profiles, and dispatch events are stored in one schema. Reporter identities (callsign-to-person mappings) are stored in a separate, isolated schema. A full export of the sighting data reveals zero real identities. The separation is structural, not policy.

Who can access the TRACE URL?

The Vercel URL (e.g. trace-abc123.vercel.app) is not indexed by search engines or listed in any directory. Only people you share it with can find it. The reporter app requires a valid invite code to access any functionality. The operator console requires a callsign and access code. An unauthenticated visitor sees a login screen and nothing else.

How do I shut everything down and delete my data?

Three steps remove all data from all servers:

1. Delete your Neon project. go to console.neon.tech, click your project → Settings → Delete Project. This permanently destroys all sighting data, vehicle records, and reporter accounts.

2. Delete your Vercel project. go to vercel.com/dashboard, click your project → Settings → scroll to bottom → Delete Project. Your website goes offline immediately.

3. (Optional) Delete the GitHub repo. go to github.com, open your TRACE repository → Settings → scroll to bottom → Delete this repository.

After these three steps, no trace of your chapter exists on any server. Reporter apps will stop working immediately and show an error screen.

Do any of these services cost money?

All three services (Neon, Vercel, GitHub) have generous free tiers. A chapter with up to ~50 reporters and thousands of sightings will stay within free limits. None of them require a credit card to sign up. If you ever approach a limit, the service will warn you before charging anything.
